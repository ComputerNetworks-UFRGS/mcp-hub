# TC003 — Products: Excessive Memory Usage

| Field            | Value                          |
|------------------|--------------------------------|
| **Fault type**   | Code bug                        |
| **Service**      | `products`                     |
| **Image tag**    | `minimal-boutique-products:tc003`|
| **Introduced by**| Source code change             |

---

## Fault description
The products source code is changed to artificially use more RAM as time passes. 
Every time a user requests GET /products or /products/\<int:product_id\>, a lot of data is stored in a list, which never gets emptied.
By setting a memory limit for the container, the pod occasionally exceeds the limit and is killed and restarted. 
The number of replicas is reduced to 1 to speed up the process.

---

## Setup
```bash 
kubectl set image deployment/products products=ghcr.io/computernetworks-ufrgs/minimal-boutique-products:tc003 -n minimal-boutique

kubectl set resources deployment products --containers=products --limits=memory=1Gi -n minimal-boutique

kubectl scale deployment/products --replicas=1 -n minimal-boutique
```

---

## Teardown

```bash
kubectl set image deployment/products products=ghcr.io/computernetworks-ufrgs/minimal-boutique-products -n minimal-boutique

kubectl set resources deployment products --containers=products --limits=memory=0 -n minimal-boutique

kubectl scale deployment/products --replicas=3 -n minimal-boutique
```


---

## Example prompts

- Why does the products service keeps restarting?

---

## Expected agent response

The agent should be able to identify that the products deployment is using more memory than usual and that it is being restarted for exceeding the memory limit.
