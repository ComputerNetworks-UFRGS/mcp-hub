# TC002 — MCP: Incorrect ports in the deployment file.

| Field            | Value                          |
|------------------|--------------------------------|
| **Fault type**   | Wrong Config                   |
| **Service**      | ` k8s-mcp-port`                |
| **Image tag**    | ` k8s-mcp-port:tc002`          |
| **Introduced by**| Source config change           |

---

## Fault description

When deploying the application with all ports swapped, and the port running the service is not the same as the port actually running MCP, it's impossible to connect to MCP.

To find the error, you should look at the pod logs to find out which port MCP is running on and set it to match in the service.

---

## Setup
- Inside the cluster Download or create Deployment-k8s-with-port-error.yaml file, locate on https://github.com/ComputerNetworks-UFRGS/mcp-hub/blob/main/k8s-mcp/Deployment-k8s-with-port-error.yaml , 

-Run
```bash
kubectl apply -f Deployment-k8s-with-port-error.yaml
```

Inside your MCP client tool try to connect to http://k8s-mcp-port.imsilva:8080/mcp and you will see a error message when try connect
---

## Teardown

```bash
kubectl delete -f Deployment-k8s-with-port-error.yaml
```

---

## Expected agent response

The agent should be able to identify the error as a code error and suggest a change to the hardcoded URL.
