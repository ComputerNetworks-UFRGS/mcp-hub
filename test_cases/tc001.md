# TC001 ŌĆö Frontend: hardcoded `host.docker.internal` API URL

| Field            | Value                          |
|------------------|--------------------------------|
| **Fault type**   | Code bug                       |
| **Service**      | `frontend`                     |
| **Image tag**    | `minimal-boutique-frontend:tc001`|
| **Introduced by**| Source code change             |

---

## Fault description

The frontend's API base URL is hardcoded to `http://host.docker.internal:<port>`.
This hostname is only resolvable in local Docker Desktop environments and fails
inside Kubernetes, causing all API calls from the frontend to fail at runtime.

---

## Setup
- Run 
```bash 
kubectl set image deployment/frontend frontend=ghcr.io/computernetworks-ufrgs/minimal-boutique-frontend:tc001 -n minimal-boutique
```
- Access the frontend in order to generate some error logs.

---

## Teardown

```bash
kubectl rollout undo deployment/frontend -n minimal-boutique
```

---

## Expected agent response

The agent should be able to identify the error as a code error and suggest a change to the hardcoded URL.
